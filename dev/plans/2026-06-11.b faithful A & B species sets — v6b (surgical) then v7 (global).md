# Plan: faithful A & B species sets — v7 (surgical) then v8 (global)

## Context
The current v6 pipeline silently bakes a **Program-Area overlap requirement into `is_ok`**
(merge_models.qmd:858), conflating "valid species" with "species in a BOEM Program Area." This collapses
the true study-area biodiversity from ~16,158 valid species to 9,424, and makes the apps/Report unable to
faithfully show life **outside** the 20 Program Areas — even though that data already exists in the DB.
The user wants to re-open querying to the full study area so mapsp/mapgl default to "show everything,"
and to do so properly by going **global** in a fresh v8: re-ingest all inputs at worldwide extent
[-180,180]×[-90,90], extend the grid to shore (globe − land), adopt global biogeographic regions
(Costello 2017 marine realms) for areas outside the BOEM US ecoregions, enable endemism/range statistics, migrate
the Species app onto the titiler tile service, and add cross-version (v1..v7) species comparison.

Delivered in two stages: **(1) v7** — drop the Program-Area gate on the existing v6 data and rescale over
the full valid set, exposing Set A (no re-ingest, but scores drift so it's a real version bump);
**(2) v8** — the full global re-ingest. Outcome:
both **A** (entire study area / global biodiversity) and **B** (within the 20 Program Areas) are produced
faithfully in the app and the Report function (mapgl → api/), with `is_ok` meaning pure validity and all
region scoping handled spatially.

## Findings (v6 review)

### What determines `is_ok = TRUE` (merge_models.qmd ~L760-875)
Three stacked gates, applied per taxon:
1. **Taxonomic/data-quality** (case_when, L766-817, separate for birds=botw vs worms): has `taxon_id`,
   has `mdl_seq`, `redlist_code != 'EX'`, `worms_is_marine != FALSE`, `worms_is_extinct != TRUE`,
   worms `taxonomic_status` in {accepted, alternative representation}, `sp_cat != 'reptile'` (turtle ok).
2. **Merge produced ≥1 cell** → assigns `mdl_seq`. Species whose IUCN range is entirely outside the US
   EEZ get a 0-cell `rng_iucn` backfill (merge_models_prep.qmd ~L3539) → merge clips to empty → 0 cells →
   no `mdl_seq` → fails gate 1's `is.na(mdl_seq)` check. (This is the IUCN-outside-EEZ exclusion.)
3. **Program-Area overlap** (L829-859): `is_ok <- is_ok & taxon_id %in% taxa_in_pra`, where `taxa_in_pra`
   = taxon_model → model_cell → zone_cell → zone(`fld='programarea_key'`, table `ply_programareas_2026_v6`).
   THIS is the unexpected filter. It uses **source/contributing models** (taxon_model), not the merged model.

### The numbers (grounded by read-only queries on v6 sdm.duckdb)
- Total taxa: 17,561.
- **Set A** = taxonomically valid + has a merged distribution anywhere in the study-area grid (gates 1-2,
  NO program-area gate) = **16,158**. ≈ the "~17,000" originally remembered.
- `is_ok = TRUE` today = **9,424** (gates 1-2-3). The PRA gate drops **6,734** valid species
  (fish 3,223; invert 2,877; bird 273; other 215; coral 130; mammal 16).
- Of the 6,964 `is_ok=FALSE` taxa that DO have a merged model, only 108 reach a PRA (excluded taxonomically);
  ~6,856 are excluded solely for being outside the 20 PRAs.
- **Set B** = merged distribution actually within the 20 PRAs = **9,230** (mapped/scored extent).
- Study-area grid = Bio-Oracle depth masked to `ply_planareas_2025` + 10 km buffer (~661k cells), spanning
  12 ecoregions incl. Atlantic (NECS/SECS), Caribbean (PUR/CAC), Pacific Islands (PIS/HAR) — excluded from
  the current program cycle (Alaska/Pacific/Gulf). **Set A data already exists in v6 `model_cell`; no re-ingest.**

### Where the PRA-only assumption is baked in downstream
- `zone_taxon` / `zone_metric` precomputed ONLY for program areas + subregions (calc_scores.qmd ~L2515:
  `filter(tbl %in% c(tbl_sr, tbl_pra))`); NOT ecoregions / full study area.
- `msens::species_for_cells()` (msens/R/calc.R:171-209) filters `taxon` by `is_ok` (L174) — so once `is_ok`
  is redefined, AOI/report species lists automatically widen to Set A. Good leverage point.
- API report (api/report.qmd): `kind="pra"` → precomputed zone lookups (Set B); `kind="wkt"` → live
  `cells_in_polygon()` over the FULL grid + `species_for_cells()` (already Set A for arbitrary polygons).
- mapsp dropdown = `taxon` filtered `is_ok` (widens automatically); default view already ecoregion-bbox.
  Misleading "outside BOEM Program Areas" empty/error messages (mapsp app.R ~L538, L913).
- mapgl: "FULL" subregion falls back to "USA" `zone_taxon` row; "cells outside Program Areas" gray overlay
  always-on; subregion/PRA species tables read PRA-only `zone_taxon`; cell-click path already live `model_cell`.

### Round-2 findings (global v7)
- **is_ok audit**: only the program-area narrowing line conflates concepts. Taxonomic gates
  (merge_models.qmd:766-817) are correct and stay. Every downstream SCORE computation is
  spatially scoped by joins (cell_metric ← model_cell; zone_taxon/zone_metric ← zone_cell), so
  widening is_ok does NOT distort within-PRA calcs — it just lets the spatial query set scope.
  App dropdowns / `msens::species_for_cells()` (calc.R:174) filter is_ok → they auto-widen to Set A.
- **Ingest extents**: AquaMaps (global 0.5° HCAF), IUCN worms-marine + BirdLife/botw (global) →
  can go global directly. NMFS CH, FWS CH, FWS range maps are **USA-only** → remain regional subsets
  (flag, don't drop). Cell grid today = Bio-Oracle global 0.5° masked to planareas+10km (661k cells);
  global 0.05° ocean-to-shore ≈ 15-20M cells (storage/perf implications).
- **FAO Fishing Areas already local** (`~/My Drive/projects/msens/data/raw/marineregions.org/fao/World_Fao_Zones.shp`) —
  considered, but **superseded by Costello 2017 realms** as the v8 global layer (see decisions).
- **to-shore**: show_study-area.qmd builds ply_planareas_2025_toshore.gpkg via Voronoi extension of PA
  boundaries to a ~12nm land buffer; reusable, swap US EEZ/land for global.
- **titiler** (server/titiler/factory.py): serves XYZ PNG from DuckDB cell_metric over a cell-id COG;
  bounds read dynamically from the COG ⇒ global COG ⇒ global tiles, no service change. mapgl uses it
  (cell_sql/cell_tile_url/add_cell_tiles). **mapsp does NOT** — pulls full model_cell into R as a terra
  raster + base64 (won't scale globally); needs a per-species (mdl_seq) tile factory + stats endpoint.
- **versions**: v1..v6 sdm.duckdb exist; taxon schema drift (v1 per-dataset cols → v3 ms_merge → v6
  is_er_spatial); all share scientific_name/worms_id/sp_cat/is_ok ⇒ cross-version compare is feasible.
- **Correction to this session's earlier docs edits**: they set "A = 9,424 across the study area",
  but 9,424 already bakes in the PRA filter. True study-area A ≈ 16,158 (US extent). Those docs (and the
  deck's number) must be revisited once v7 separates validity from PRA membership.

## Proposed approach — two stages: v7 (surgical) then v8 (global)

Core principle (both stages): **`is_ok` = taxonomic/data validity + has a modeled distribution in the
study area (US extent in v7, global in v8). Program-Area / region membership is a SPATIAL query (zones),
never a taxon flag.** Set A (study area) and Set B (Program Areas) become the same query at different zone
scopes. No `is_in_pra` column needed.

## STAGE 1 — v7: remove the Program-Area gate + rescale over the full valid set (no re-ingest)
Goal: faithfully expose **Set A (~16,158, US study area)** and **Set B (~9,230, Program Areas)** from data
already present in v6 — the merged `model_cell` for the dropped species already exists; only the `is_ok`
flag hides them. No re-ingest, but scores DO change (see 1.3), so it is a real version bump (v6→**v7**, not
a "v6b" point release).

- **1.1 Recompute `is_ok`** (merge_models.qmd): delete the program-area narrowing (L829-859 `taxa_in_pra`
  + `is_ok & taxon_id %in% taxa_in_pra`). `is_ok` = taxonomic gates (L766-817) + `mdl_seq` not null. No
  merge re-run needed — apply as a targeted `UPDATE taxon SET is_ok = …` on a copy of v6 → new `v7`
  sdm.duckdb. Verify the count flips ~9,424 → ~16,158.
- **1.2 Precompute Set A zones** (calc_scores.qmd ~L2495/2515): extend `zone_taxon`/`zone_metric` beyond
  `{subregion, PRA}` to the 12 US ecoregions (already in `zone`/`zone_cell`) + a study-area-wide aggregate
  zone, so Set A species lists/scores are precomputed. Keep PRA + subregion for Set B.
- **1.3 Rescaling — RESOLVED (option b):** recompute the per-ecoregion min/max rescaling over the **full
  valid (Set A) species set**, not the old PRA-narrowed set. Within-PRA raw `cell_metric` is unchanged
  (added species have no cells inside PRAs), but rescaled scores shift where added species share a US
  ecoregion. This is the substantive change that makes it v7 (not a v6b point release). Diff v7-vs-v6
  `zone_metric` for the 20 PRAs and **report the (expected small) PRA score delta**.
- **1.4 Apps (US extent, stay on current rendering — no titiler migration yet):** mapgl/mapsp default to
  the full study area ("show everything"); dropdowns now Set A; make the "cells outside Program Areas"
  overlay optional; fix the FULL→USA `zone_taxon` fallback; fix misleading "outside Program Areas"
  empty/error copy (mapsp app.R ~L538, L913). Point apps at `ver <- "v7"`.
- **1.5 API / Report:** `msens::species_for_cells()` (calc.R:174 filters `is_ok`) auto-widens to Set A;
  add explicit A-vs-B side-by-side (full biodiversity vs Program-Area-overlap subset) in report.qmd /
  report_area_child.qmd.
- **1.6 Docs:** correct the narrative — **A ≈ 16,158** (US study area), **B ≈ 9,230** (Program Areas);
  fix the `is_ok` definition (schema.qmd) and **revise this session's earlier edits** (index/intro/summary/
  science/data-sources) and the tour deck number, which wrongly used 9,424 as "A".

Deliverable: a `v7` database + apps/report that show true biodiversity everywhere in the US study area,
with Program Areas as one selectable scope, plus a documented PRA score delta.

## STAGE 2 — v8: global re-ingest (builds on v7's validity-only `is_ok`)
Everything below goes worldwide; carries forward the v7 principle that region scope is spatial, not a flag.

### Phase 0 — Decisions (RESOLVED) & scaffolding
- Extent: global **[-180,180]×[-90,90]**; standardize longitude on **[-180,180]** everywhere (today the
  cell COG is 0-360 — reproject COG + ingests + apps to match).
- Rescaling regions = **two overlapping layers**: keep the **BOEM USA Ecoregions** where they exist; use
  **Costello et al. (2017) Marine Biogeographic Realms** for everywhere else. Because a realm overlaps the
  BOEM ecoregions, a realm's rescale min/max is computed over its **full** extent (incl. the BOEM overlap),
  so this is NOT a single non-overlapping partition — a cell can carry both a BOEM-ecoregion score and a
  realm score. (Costello shapefile: figshare 5596840 — see reference memory.)
- US/PRA scores are **allowed to drift** under the consistent global framework (no back-compat constraint).
- Grid: **globe − Natural Earth land (high-res); any cell with ≥1% ocean overlap counts as ocean** (this is
  how we reach "to shore" — no Voronoi for the grid). 0.05° resolution. `ver <- "v8"` in libs/paths.R.

### Phase 1 — Global ocean cell grid
- Generalize ingest_aquamaps_to_sdm_duckdb.qmd grid build (L503-578): replace the planarea mask with a
  global 0.05° grid masked by **(globe − Natural Earth hi-res land)**, keeping any cell ≥1% ocean. Emit a
  global cell-id COG in [-180,180] (`r_cellid_global.tif`). Record per-cell area_km2; keep cell_id stable
  for the titiler COG↔DuckDB mapping.

### Phase 2 — Global data ingest → model / model_cell / taxon
- Re-run AquaMaps, IUCN (worms-marine), BirdLife/botw ingests at global extent (remove US clipping).
- Keep NMFS/FWS (US-only) as-is; add a dataset-level `extent` field (global vs USA) in `dataset` for
  honesty in the apps/report. Be explicit about each input's native extent.
- Rebuild taxon from the (global) merge. Expect Set-A-scale taxon counts (≫ 9,424).

### Phase 3 — is_ok decoupling + two-layered ecoregions
- merge_models.qmd: same decoupling as v7 (no program-area narrowing); is_ok = validity + has merged cells
  in the global study area. Update schema.qmd is_ok description.
- Build zones into zone/zone_cell: **BOEM USA Ecoregions** + **Costello realms** (the two rescale layers,
  overlapping), plus BOEM Program Areas (ply_programareas_2026) + subregions. Use a nearest-neighbour /
  Voronoi fill ONLY to guarantee **every ocean cell is attributable to an ecoregion/realm** (no
  unattributed cells) — not for the grid mask. Program-area membership = zone query.

### Phase 4 — calc_scores (global, two-layered rescaling)
- zone_taxon/zone_metric (calc_scores.qmd ~L2495,2515): widen the zone set to BOEM USA ecoregions +
  Costello realms (+ PRA, subregion) so Set A lists/scores are precomputed globally; live model_cell stays
  for arbitrary AOIs.
- Rescaling: a cell inside a BOEM USA ecoregion is rescaled within that ecoregion; cells elsewhere are
  rescaled within their Costello realm, where realm min/max spans the realm's **full** extent (incl. any
  BOEM overlap). US/PRA scores will drift vs v6/v7 — quantify and document, don't constrain.

### Phase 5 — titiler global + species tiles
- Point titiler at the global cell-id COG + v8 DuckDB (env vars only; bounds auto-detected).
- Add a species tile factory in factory.py keyed by `mdl_seq` over `model_cell` (mirror the cell_metric
  factory), plus `/species/statistics`. This unblocks global mapsp.

### Phase 6 — apps
- mapgl: default scope = global "show everything"; Program Areas an optional focus; make the
  "cells outside Program Areas" overlay optional; fix FULL→USA zone_taxon fallback to a true global row.
- mapsp: migrate get_rast() (app.R:725-762) from in-R terra raster → titiler species tiles
  (msens::add_cell_tiles + species tile URL); default global; fix misleading "outside Program Areas"
  empty/error copy (L538, L913).

### Phase 7 — API / Report (A and B)
- species_for_cells/scores_for_cells already compute per-AOI from model_cell/cell_metric; with widened
  is_ok they yield Set A automatically. Add explicit A-vs-B reporting: for any area, report full
  biodiversity (A) and the Program-Area-overlap subset (B) via a zone-intersect, side by side in
  report.qmd / report_area_child.qmd. Add the **endemism / range-size** stat (global range enables it —
  pairs naturally with the Costello realms, which were derived for species endemicity).

### Phase 8 — cross-version species comparison
- Notebook/function comparing taxon across v1..v8 sdm.duckdb on worms_id/scientific_name: counts by
  sp_cat, is_ok membership churn, added/dropped species per version. Account for schema drift (v1 lacks
  extrisk/ms_merge). Output a tidy comparison table + figure.

### Phase 9 — docs
- Rewrite the species-count narrative: A = study-area/global biodiversity (~16k US in v7 / larger global in
  v8), B = ~9.2k within the 20 Program Areas; correct the is_ok definition; revise the earlier-edited docs
  (index/intro/summary/science/data-sources) and the tour deck number once v7 numbers are final.

## Open design decisions — RESOLVED
- **v7 / 1.3**: rescale over the full valid set (option b); US/PRA scores may drift; report the delta.
- **v8 / Phase 0**: (1) rescaling = two overlapping layers — BOEM USA Ecoregions + Costello (2017) realms
  outside, realm stats computed over the full realm extent; (2) longitude standardized to [-180,180];
  (3) US/PRA scores allowed to drift for a consistent framework; (4) grid = globe − Natural Earth hi-res
  land, any cell ≥1% ocean.
- Residual watch-item (not blocking): DuckDB size / tile-cache budget at the global cell count.
